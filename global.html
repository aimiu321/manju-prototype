全局模块 · 登录与积分与会员 交互说明

一、登录

项目说明
登录方式手机号 + 短信验证码（11位手机号 + 4位以上验证码）
触发时机仅在触发需积分的AI操作时校验登录
未登录流程弹出登录弹窗 → toast「您还未登录」→ 登录成功 → 关闭弹窗 → 不自动继续操作

二、积分体系

积分来源

类型说明有效期
充值积分购买积分包或会员订阅获得永久有效
赠送积分活动赠送、新人注册、会员额外赠送当期有效，过期清零
系统补偿AI生成失败/异常补偿有有效期，过期清零
消费优先级：赠送积分 > 系统补偿积分 > 充值积分（优先消耗即将过期的积分）

积分消耗标准

环节操作积分冻结时机扣减/解冻
环节1AI优化/灵感/拆解/重新分析2点击时冻结成功扣减；失败退还
环节2角色图生成/重新生成10/张点击时冻结成功扣减；失败退还
环节3场景图生成/重新生成10/张点击时冻结成功扣减；失败退还
环节4分镜视频生成/重新生成50/条点击时冻结成功扣减；失败退还

一键生成批量冻结

一键生成时，按「待生成数量 × 单价」一次性冻结总额。每张/条独立生成，成功扣减对应积分，失败退还对应积分，互不影响。

三、积分不足半弹窗

项目说明
触发条件积分余额 < 操作所需积分
样式底部半屏弹窗，显示当前积分/所需积分
非会员按钮「开通会员」→ 会员订阅Tab
会员按钮「去充值」→ 积分充值Tab
充值后行为回到当前页面，不自动继续操作

四、首次生成确认弹窗

场景是否弹窗内容
首次生成弹出确认弹窗「本次生成将消耗 N 积分」
重新生成无弹窗，直接执行—
一键生成弹出确认弹窗「本次将生成 N 张/条，共消耗 N 积分」

六、Apple支付说明

八、调试面板说明
