环节5 - 合成视频 操作说明
1页面结构
Page1 合成视频 视频播放区 + 分镜组成（可折叠）+ 底部操作栏
分镜选择半屏 点击"选择分镜"/"重新合成"弹出，选择≥2个分镜后合成
分镜组成 视频区下方可折叠区域，展示合成视频包含的分镜信息
"我的漫剧"（剧本列表+剧集列表+创建下一集）为全局模块，独立于环节5
2不同入口进入逻辑
从环节4进入 → 若已有mergedVideoUrl直接展示，否则显示引导态"点击下方选择分镜开始合成"
从我的漫剧进入 → 直接加载已保存视频，隐藏"选择分镜"和"保存剧集"按钮
调试器"入口来源"可切换模拟不同入口
3合成状态
空闲 显示引导态"点击下方选择分镜开始合成"，底部"选择分镜"按钮
合成中 Loading动画 + "正在合成视频..." + "分镜 N/M 已处理"
合成完成 视频播放器（首帧+播放按钮+时长标签）
合成失败 ❌ + "合成失败，请重试" + "重新合成"按钮
4底部按钮逻辑（动态显隐）
选择分镜 从环节4进入且未合成时显示，合成中隐藏
重新合成 合成完成/失败时显示（从环节4进入），合成中隐藏
下载视频 始终显示，合成完成时可用
保存剧集 仅从环节4进入+合成完成时显示，点击后变为"保存中..."loading
重新合成（失败） 合成失败时替换"保存剧集"位置
从我的漫剧进入时，只显示"下载视频"
5返回逻辑
合成完成/空闲 → 直接返回环节4
合成中 → 弹窗「视频正在合成中，离开将中断合成，确定返回吗？」→ 确定 → 取消合成任务 → 返回环节4
6分镜选择半屏
展示所有分镜，已生成(videoNeedsRegen=false)可勾选，其余置灰
默认选中所有可选分镜
≥2个分镜时"合成"按钮可用
点击"合成"→传入shotIds→关闭半屏→开始合成
合成顺序固定按分镜序号，MVP不支持拖拽调序
7保存剧集
点击"保存剧集" → 校验登录 → 按钮变为"保存中..." → 保存成功半屏
保存失败 → 按钮恢复"保存剧集" + toast"保存失败，请重试"
半屏中可编辑剧集名称（默认"第N集"，最大20字）
保存操作在弹出半屏前已完成，"查看我的漫剧"不再触发保存
已保存剧集永久保留云端，不受分镜7天限制
8视频播放
iOS原生AVPlayer控件：播放/暂停/进度条/音量/全屏
点击视频区域 → 播放；再次点击 → 暂停
播放完毕回到首帧
下载视频 → 请求相册权限 → 保存到相册；权限拒绝→弹窗引导去设置
9积分与登录
合成视频：0积分（FFmpeg服务端处理，无AI成本）
保存剧集：0积分（云存储操作）
保存剧集需登录（关联用户剧集库）
下载视频无需登录
查看我的漫剧需登录
10关键设计决策
分镜选择移入环节5 → 合成视频是环节5核心操作，选择+合成紧耦合
shotIds客户端传入（不持久化）→ 每次合成独立选择，无需selectedShotIds字段
服务端拼接（非客户端）→ 视频已在云端，避免客户端集成FFmpeg
FFmpeg -c copy优先 → Seedance输出编码一致，快且无损
最低2个分镜 → 1个无需合成
重新合成覆盖旧结果 → 同一episode不同版本，覆盖比新建合理
剧集永久保存 → 最终成品不受分镜7天限制
MVP不支持删除剧集 → 避免误删，减少复杂度
保存先于半屏确认 → 保存完成后才弹半屏，半屏仅编辑名称
并发合成防护 → merging状态隐藏选择/重新合成按钮
轮询超时2分钟视为失败 → 避免无限等待
