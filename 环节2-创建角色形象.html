环节2 · 创建角色形象
1页面结构
一级页面 角色卡片列表，每张卡片含角色图区+角色信息+状态标签
二级页面 点击卡片进入，含模式切换、描述/外貌编辑、参考图上传、生成按钮
角色库 半屏弹窗，展示本剧已生成角色，支持引入到当前集
所有交互均为前端模拟，AI返回预设demo数据
2一级页面操作
点击卡片区域 → 进入二级页面编辑角色
点击 生成形象 按钮 → 校验登录+积分 → 积分充足直接执行（无确认弹窗）
点击 重新生成 按钮 → 积分充足直接执行（无确认弹窗）
点击已生成角色图 → 全屏放大预览，点击空白关闭
点击 角色库 → 半屏弹出，3列网格展示本剧已有角色
点击 重新分析 → 确认弹窗（消耗2积分），全页等待遮罩，拆解完成后刷新角色列表
3二级页面操作
模式切换 点击「文生图」/「文+参考图」标签切换生成模式
参考图上传：模式2下点击+号 → 选择照片 → 预览缩略图
参考图替换/删除：点击已上传缩略图 → 弹出选择「拍照/从相册选择/删除」
描述/外貌编辑 点击文本进入编辑态（紫色边框），描述≤500字，外貌≤80字
编辑已生成角色 → 自动标记 需重新生成
放大预览：点击已生成角色图 → 全屏查看
点击 ←返回 → 直接返回一级页面（无确认弹窗）
4角色图状态
待生成 占位图+「生成形象」按钮
生成中 骨架屏动画，按钮隐藏
已生成 角色图展示+「重新生成」按钮+可放大预览
已选择 角色图（来自角色库引入/自动填充）+「重新生成」按钮
生成失败 ❌图标+「重新生成」按钮+退还积分
5底部按钮逻辑
重新分析 消耗2积分+确认弹窗，全页等待遮罩后刷新角色列表
一键生成 并行后台生成所有待生成角色，每个独立完成后toast通知
下一步 全部角色已生成后显示，跳转环节3
6角色库规则
角色库按剧（series）隔离，不同剧集互不可见
角色图生成后自动存入角色库
非第一集：自动从角色库匹配填充（状态「已选择」）
手动引入：点击角色库 → 选择引入 → 带入角色图+描述+外貌
同名角色不可重复引入，按钮变为「已选择」
7前序环节联动
画风变更 → 所有已生成角色标记 需重新生成
原因：画风变更后角色图视觉风格不一致
调试面板点击「模拟画风变更」可验证此联动效果
8相册权限
首次上传 → 弹出系统权限请求「允许访问照片？」
已拒绝 → 弹出引导弹窗「无法访问相册」→ 点击「去设置」跳转系统设置
返回后重检：从设置返回时自动重新检查权限状态
调试面板可手动切换「未请求/已授权/已拒绝」状态
9返回与导航
一级页面←返回 → 确认弹窗「返回后剧本将重新拆解，确认返回吗？」
二级页面←返回 → 直接返回一级页面（无确认弹窗）
返回按钮独立于重新分析，两者互不影响
10积分与登录
角色图生成：10积分/张，生成/重新生成/一键生成均需消耗
重新分析：2积分/次（唯一需确认弹窗的操作）
校验流程：未登录→登录弹窗 → 积分不足→积分不足半屏 → 充足→直接执行
积分不足半屏：非会员→「开通会员」，会员→「购买积分包」
生成成功扣减积分，失败/超时解冻退还
生成/重新生成无确认弹窗，积分充足直接执行
